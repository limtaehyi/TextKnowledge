## 1. 부적절한 자격 증명 사용(Improper Credential Usage)  
<br>

- 모바일 앱에서 부적절한 자격 증명 관리로 인한 보안 위험
예: 평문 저장, 약한 암호화, 부적절한 인증 메커니즘 등
  
<br>
<br>

## 2. 불충분한 공급망 보안(Inadequate Supply Chain Security)  
<br>

- 모바일 앱의 공급망 보안 취약점으로 인한 위험
예: 취약한 오픈소스 라이브러리, 악성 SDK, 부적절한 코드 서명 등
  
<br>
<br>

## 3. 보안에 취약한 인증 및 권한 부여(Insecure Authentication/Authorization)  
<br>

- 모바일 앱의 인증 및 권한 부여 메커니즘의 보안 취약점
예: 약한 암호, 부적절한 세션 관리, 부족한 다단계 인증 등
  
<br>
<br>

## 4. 부족한 입력/출력 검증(Insufficient Input/Output Validation)  
<br>
- 모바일 앱의 입력 및 출력 데이터 검증 부족으로 인한 보안 위험
예: SQL injection, 크로스 사이트 스크립팅(XSS), 버퍼 오버플로우 등
  
<br>
<br>

## 5. 취약한 암호화(Insecure Cryptography)  
<br>

- 모바일 앱의 암호화 메커니즘 취약점으로 인한 보안 위험
예: 약한 암호화 알고리즘, 부적절한 키 관리, 부족한 난수 생성 등
  
<br>
<br>

## 6. 부적절한 보안 구성(Insecure Security Configuration)  
<br>

- 모바일 앱의 보안 구성 설정 미흡으로 인한 보안 위험
예: 기본 설정 사용, 불필요한 기능 활성화, 부적절한 권한 관리 등
  
<br>
<br>

## 7. 취약한 데이터 보호(Insecure Data Protection)  
<br>

- 모바일 앱의 데이터 보호 메커니즘 취약점으로 인한 보안 위험
예: 민감 데이터 평문 저장, 부적절한 데이터 백업 및 복구 등
  
<br>
<br>

## 8. 부족한 보안 모니터링 및 로깅(Insufficient Security Monitoring & Logging)  
<br>

- 모바일 앱의 보안 모니터링 및 로깅 기능 부족으로 인한 보안 위험
예: 부족한 보안 이벤트 탐지 및 대응, 부적절한 로그 관리 등
  
<br>
<br>

## 9. 취약한 코드 품질(Insecure Coding Practices)  
<br>

- 모바일 앱 개발 시 보안 코딩 원칙 미준수로 인한 보안 위험
예: 버그, 취약점, 보안 모범 사례 미준수 등
  
<br>
<br>

## 10. 부족한 보안 위험 관리(Insufficient Security Risk Management)  
<br>

- 모바일 앱 보안 위험 관리 프로세스 부족으로 인한 보안 위험
예: 보안 위험 평가 미흡, 보안 대책 수립 및 이행 부족 등
