1. DB
 
 DB, DBMS
 - 데이터베이스란 여러 데이터를 저장하기 위한 집합체를 뜻한다.
 - 데이터베이스를 관리하고 운영하는 소프트웨어를 DBMS (DB Management Sys)라고 하며 데이터를 저장하는 방식과 시스템에 따라 종류가 갈린다.
 - MySQL, MariaDB, PostgreSQL, Oracle, SQL Server, DB2, Access, SQLite 등등


2. 계층형 DB

 HDBMS
 - 폴더와 파일등의 계층 구조로 데이터를 저장하는 방식
 - 데이터와의 관계를 트리 구조로 정의 하여 부모-자식 형태를 가진다.
 - 종류로는 하드디스크, DVD 파일 시스템
 - 장점은 데이터의 엑세스 속도가 빠르고 데이터 사용량을 쉽게 예측할 수 있다.
 - 단점은 상하 종속적인 관계로 초기 세팅후 프로세스 수용이 어려울 수 있다.


3. 관계형 DB 

 RDBMS
 - 행, 열을 가지는 표형식으로 데이터를 저장하는 방식
 - SQL을 이용하여 관리 및 접근
 - 종류로서는 oracle, sql server, postgresql, mysql이 있다.
 - 장점으로는 다양한 용도 가능, 높은 성능, 데이터 무결성 보장, 높은 신뢰성, 명확한 스키마 정의, 관계를 통한 데이터를 중복없이 한번만 저장 한다.
 - 단점은 대량의 데이터 입력 처리, 컬럼 확장 어려움(유연성 부족), 수평 확장의 어려움 


4. NoSQL

 Key-value
 - 키-값 형태로 저장되는 데이터 베이스이며 python의 dictionary 형태로 저장하는 방식
 - join이 없고 데이터를 가져올 때 컬렉션에 있는 데이터를 복제하여 필요한 데이터 일부만 가져옴
 - 정확한 데이터 요구 사항을 알 수 없거나 관계를 맺는 데이터가 자주 변경될때 사용
 - RDB와 다르게 비정형 데이터(메신저 텍스트, 음성, 이미지, 영상 등)을 다룰 수 있음, 분산형 컴퓨팅에 적합
 - 장점으로는 스키마가 없기 때문에 유연하며, 언제든지 저장된 데이터를 조정하고 새로운 필드를 추가 할 수 있고 데이터는 앱이 필요로 하는 형식으로 저장되어 데이터 읽어오는 속도가 빠름, 수평 및 수직 확장에 용이함
 - 단점으로는 과한 유연성으로 데이터 구조를 결정하지 못할 수 있음, 데이터 중복은 여러 컬렉션과 문서가 여러개의 레코드에 변경된 경우 업데이트를 해야함, 데이터가 여러 컬렉션에 중복되어 있어 수정을 해야 하는 경우 모든 컬렉션에서 수정해야 함
 -  종류는 redis, oracle berkely가 있음

 Document
 - XML, JSON 계층적 트리 데이터 방식으로 저장
 - 집합적 데이터 모델로 관계형 DB에서 여러개의 테이블 데이터를 하나의 document에 모아 둘 수 있음
 - 조회시 한번 조회로 필요한 데이터 획득 가능, join 기능을 대체
 - Key-value와의 차이는 db에 값을 문서로 저장함
 - 트렌지션이 원자적으로 이루어지고 데이터를 여러 서버에 분산 저장이 가능하고 복제, 회복이 가능함
 - 종류로는 MongoDB, Azure cosmos가 있음


5. DB 분할

 수평 분할
 - 수평 분할은 테이블을 행 단위로 나누는 것을 의미하며 각각의 파티션들을 샤드라고 부른다.
 - 예를 들어 DB를 수평 분할 할 경우 sharding(분할 알고리즘)을 통해 shard들과 각각 shard key들이 주어진다.
 - 장점으로는 성능 상향, 확장성, 보안, 유지보수의 쉬움이 있다.
 - 단점으로는 데이터 중복, 쿼리 복잡성 증가, 파티션간의 균등 분산의 어려움
 - 데이터 무결성 관리(일관성)이 어려움

 수직 분할
 - 수직 분할은 테이블을 열 단위로 나누는 것을 의미하며 스키마를 나누고 데이터가 따라 옮겨가는 것을 말한다.
 - primary key를 복사하여 값들을 나누며 보통 id를 복사하여 분류한다.
 - 장점으로는 자주 사용하는 열만 가져오니 성능 개선, 보안 강화, 쉬운 유지보수가 있다.
 - 단점으로는 쿼리가 복잡성, 중복 데이터, 확장성 제한이 있다.


6. SQL

 SQL
 - Structed Query Language
 - 데이터베이스에서 자주 사용하는 쿼리 언어로 DDL, DML, DCL, TCL인 순서대로 정의, 조작, 제어, 트렌젝션 제어로 4개로 나뉜다. 

 DDL
 - Create : 테이블 생성
 - Alter : 테이블 구조 수정
 - Drop : 테이블 삭제
 - Rename : 테이블 이름 수정
 - Comment : 
 - Truncate : 테이블 모든 데이터 삭제

 DML
 - Select : 데이터 검색
 - Insert : 데이터 추가
 - Update : 데이터 수정
 - Delete : 데이터 삭제
 - Merge : 데이터 체크 후 CRUD를 한번에 작업
 - Call : 사용자 정의 함수
 - Explain plan
 - Lock table

 DCL
 - Grant : 사용자에게 작업 수행 권한 부여
 - Revoke : 권한 회수

 TCL
 - Commit : 트렌잭션 작업 완료 후 데이터베이스 반영
 - Rollback : 트렌잭션 변경 사항 취소 및 이전으로 복구
 - Savepoint
 - transaction
