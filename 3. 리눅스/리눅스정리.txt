리눅스 정리

1.리눅스 개요
 - 리눅스는 윈도우,유닉스와 같은 os이다.
 - 1989년 핀란드 헬싱키 대학의 재학중인 리누스 토르발스(Linus Torvalds)가 처음으로 유닉스를 기반으로 리눅스를 개발했다.
 - CUI(Command User Interface)이며 강력한 네트워킹, 계층형 파일 시스템등 여러장점이 있다.
 - 이식성, 개발환경, 네트워크, 보안성, 하드웨어 지원, 우수한 성능, 신뢰성등이 장점이다.
 - 리눅스 종류로서는 레드헷, 데비안, 수세, 칼데라, 맨드레이크, 슬랙웨어가 있다.
 - 유저는 하드웨어를 보안, 사용최적화등 여러 요인으로 직접적으로 사용할수가 없다. 그래서 사용자가 쉘을 통해 원하는 명령을 전달, 커널을 통해 하드웨어를 사용하는 것이다.
 - 유저 -> 쉘 -> 커널 -> 하드웨어로 직렬로 통하기 때문에 분석하기 쉬운거와 다르게 윈도우는 여러 방법을 통해 하드웨어와 통신할수 있기 때문에 분석하기 어렵다.

4.리눅스 활용
 - 리눅스 파일 시스템은 ext2, ext3, ext4가 있으며 현재는 ext4를 많이 지원한다.
 - 프로세스들은 셸-커널-하드웨어-커널-셸 순으로 실행된다
 - 쉘은 bash, csh, dash, ksh, sh, tcsh, zshell이 있다.
 ㄴ sh : 최초의 유닉스 쉘로 스크립트를 지원한다. 본쉘은 논리 및 산수 연산을 처리하는 내장기능이 없어 이전에 사용한 명령어를 기억할수 없다.
 ㄴ k : 본쉘을 개선한 쉘로 history, alias등 작업기능이 추가되었다.
 ㄴ c : ksh처럼 본쉘 개선버전으로 history, alias, ~등 기능 추가됨. 명령어 편집 기능은 없음
 ㄴ tc : csh 개선 버전으로 명령어 편집기능 제공, 자동완성, where 제공
 ㄴ bash : 본쉘의 확장버전으로 만든 unix 쉘로 linux, mac의 기본 쉘로 사용됨. mv, cp, rm, touch, ls, mkdir, cd, rmdir 추가
 ㄴ z : bash,ksh,tcsh의 기능을 결합하여 맞춤법 검사, 로그인 감시, 자동생성, 플러그 및 테마 지원
 - 프로세스는 백그라운드에서 실행되는 standalone, 메모리에 상주되지 않고 클라이언트 요청시 wake up 되는
 inetd 방식이 있다.
 - init 프로세스는 /etc/inittab 파일을 읽고 해당 파일에 설정된 프로세스를 실행함
 - run level
 ㄴ 0, halt : 시스템 중지
 ㄴ 1, single user mode : 단일 사용자 모드
 ㄴ 2, multiuser, without NFS : 네트워크 사용하지 않는 다중 사용자 모드, 여러 계정으로 로그인 가능
 ㄴ 3, full multiuser mode : 네트워크 지원하는 다중 사용자 모드
 ㄴ 4, unused : 사용되지 않는 런레벨이지만 사용자가 정의해서 사용 가능
 ㄴ 5, x11 : x window를 사용하는 다중 사용자 모드(그래픽)
 ㄴ 6, reboot : 시스템 재부팅
 - 특정 목적을 가진 컴퓨터 시스템을 임베디드 시스템이라 한다.
 - 가상화 기술로 다양한 시스템을 논리적으로 하나의 시스템으로 제공하는 것을 클라우드 컴퓨팅이라 한다.

6.daemon(데몬)
 - 데몬이란 리눅스에서 프로세스인데 프로세스중에서 서버역할을 하는 것을 지칭한다.
 - 데몬은 service나 systemctl로 제어가 가능하다
 - 보통 프로세스 뒤에 d를 붙여 표현한다.
 - 데몬을 죽이는 명령어는 kill이며 killall도 있는데 kill은 PID(Proccess ID)를 개별적으로 써줘야하고 killall은 sshd나 httpd등과 같이 데몬 이름을 써줄 수 있다.


7.ch
 - ch는 권한이나 소유를 변경하는 명령어들이며 chmod, chgrp, chown이 있다.
 - chmod와 chown의 차이점은 주체이다. chmod는 파일이 사용자에게 할 수 있는 역할을 주는거고 chown은 사용자가 파일이 누구것인지 선택하는 거다.
 - chgrp은 chown의 그룹버전이다.
 - umask는 기본 권한 설정이며 777에서 설정한 값을 뺀다.
