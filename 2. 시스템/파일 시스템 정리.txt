파일시스템정리

1.FAT
 - FAT(File Allocation Table) 파일시스템은 1976년에 빌 게이츠에 의해 최초 구현되었다. 그가 구현한 목적은 자신의 회사 제품인 BASIC에서 플로피
 디스크를 관리하는데 이용하기 위해서 였다.
 - FAT은 FAT12 -> FAT16 -> VFAT -> FAT32 -> exFAT순으로 개발되었으며 PBR, Reserved, FAT1/2, Root Directory, Data Cluster, FS info, Backup Area
 로 구성되어있다.
 - 특징으로는 여러 파일시스템중 가장 간단한 구조를 가지고 있고 매우 적은 오버헤드로 시작하기 때문에 200MB미만의 드라이브/파티션에 적합하다.


2.NTFS
 - NTFS(New Technology File System)은 Windows NT로부터 00,03,08,vista,7,8,10에 이르기까지 사용되고 있다.
 - Windows NT는 개인 사용자를 겨냥하던 Microsoft사가 최초로 출시한 서버형 운영체제이다. Windows NT가 개발되면서 FAT의 구조상의 한계점을 개선하기 위한 새로운 파일시스템인 NTFS가 함께 개발되었다.
 - NTFS 1.0 -> NTFS 1.2 -> NTFS 3.0 -> NTFS 3.1 
 - MBR(Master Boot Record)는 파티션을 나눌때 생성되며 파티션정보, 부트코드, 시그니처로 구성되어있고 부트코드는 나뉜 파티션의 할당된 영역을 표시하고 시그니처는 55 AA이다.
 - MFT(Master File Table)는 NTFS에서 가장 중요한 정보로 포맷할때 생성된다. NTFS는정형화된 볼륨 구조 대신 MFT가 존재하는데 이는 볼륨에 존재하는 모든 파일과 디렉터리 정보를 담고 있다.
 - EFS(Encrypting FIle System)라고 불리는 암호화 기능이 있고 기타 유니코드 지원, LZ77의 변형 압축등이 있다.


3.EXT
 - linux를 최초 지원하는 파일시스템은 Minix의 파일 시스템이었지만 여러 문제로 Linux를 위한 EXT를 개발하게 되었다.
 - EXT2 -> EXT3 -> EXT4순으로 개발되었다. 특히 EXT4는 병렬 분산 시스템을 위해 저장공간을 늘리고 성능향상을 목적으로 3년간 개발되었다. 오늘날에도 스마트폰에서 많이 사용된다.
 - 이 파일 시스템은 블록단위로 구성되어 각각 블록 그룹은 수퍼블록과 블록그룹 디스크립터의 사본을 가지고 있으며, 모든 블록은 블록 비트맵, inode 비트맵, inode테이블, 데이터 블록을 담고 있다. 
 - 슈퍼블록은 운영체제 부팅 과정에서 필요한 정보를 담고 있으며, 블록 그룹 디스크립터는 블록 비트맵, inode 비트맵, 그리고 모든 블록 그룹 inode 시작 위치를 저장한다.
 - 모든 파일이나 디렉토리는 inode로 표현되며 inode table을 통해 관리된다.
 - 하위 호환성이 있으며 초단위인 시간소인을 사용하고 있어 정확하다. 
 - EXT4는 최대 1EBytes의 파일시스템을 제공한다. 이는 EXT3의 최대 파일 크기의 8배가 이고 서브디렉토리도 거의 무한대로 확장되었다. 디렉토리 인덱싱도 해쉬된 B트리 형태로 제한이 크게 확장되었음에도 불구하고 EXT4는 빠른 탐색이 가능하다.
